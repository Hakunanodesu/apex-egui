# 日志系统使用说明

## 概述

本项目已集成完整的日志系统，用于记录程序运行过程中的各种信息，包括调试信息、错误信息、性能数据等，方便开发者回溯和分析问题。

## 功能特性

### 1. 多级别日志记录
- **DEBUG**: 调试信息，用于开发调试
- **INFO**: 一般信息，记录程序正常运行状态
- **WARNING**: 警告信息，可能存在问题但不影响运行
- **ERROR**: 错误信息，程序运行出错
- **CRITICAL**: 严重错误，程序可能无法继续运行

### 2. 双重输出
- **文件日志**: 所有日志都会保存到 `logs/` 目录下的文件中
- **控制台日志**: INFO级别以上的日志会同时显示在程序界面上

### 3. 自动日志管理
- **自动清理**: 定期清理超过7天的旧日志文件
- **大小控制**: 当日志总大小超过100MB时自动清理最旧的文件
- **文件命名**: 日志文件按时间戳命名，格式为 `TGC_YYYYMMDD_HHMMSS.log`

## 日志文件位置

```
项目根目录/
├── logs/                    # 日志文件目录
│   ├── TGC_20241201_143022.log
│   ├── TGC_20241201_150145.log
│   └── ...
```

## 使用方法

### 1. 查看日志

#### 方法一：使用内置日志查看器
1. 在主程序界面点击 **"查看日志"** 按钮
2. 在日志查看器中选择要查看的日志文件
3. 使用过滤功能快速查找特定信息

#### 方法二：直接打开日志文件
1. 进入 `logs/` 目录
2. 用文本编辑器打开对应的 `.log` 文件

### 2. 日志查看器功能

#### 基本功能
- **文件选择**: 下拉菜单选择不同的日志文件
- **实时刷新**: 点击刷新按钮更新文件列表
- **语法高亮**: 不同级别的日志用不同颜色显示
- **滚动查看**: 支持水平和垂直滚动

#### 过滤功能
- **文本过滤**: 输入关键词过滤包含该关键词的日志行
- **级别过滤**: 选择特定日志级别（DEBUG/INFO/WARNING/ERROR/CRITICAL）
- **清除过滤**: 一键清除所有过滤条件

#### 右键菜单
- **复制**: 复制选中的日志内容
- **全选**: 选择所有日志内容
- **导出**: 将选中的内容导出为文本文件

## 常见问题排查

### 1. 程序启动失败
查看日志中的错误信息：
```
[ERROR] TGC: 加载配置文件失败: [Errno 2] No such file or directory: 'user_config.json'
[WARNING] TGC: 未检测到用户配置文件
```

### 2. 手柄映射问题
```
[ERROR] TGC: 手柄映射启动失败
[WARNING] TGC: 检测到手柄实例变动
```

### 3. 性能问题
查看性能统计日志：
```
[DEBUG] TGC: 性能统计 - 周期: 1000, 总延迟: 25.67ms, 截图: 5.23ms, 推理: 20.44ms
```

### 4. 驱动问题
```
[WARNING] TGC: 缺少驱动: ['ViGEm Bus Driver', 'HidHide']
[ERROR] TGC: 手柄独占操作失败: CM_Locate_DevNodeW failed: 0x000D
```

## 日志清理

### 自动清理
- 程序启动时自动启动日志清理服务
- 每24小时自动清理一次
- 保留最近7天的日志文件
- 总大小超过100MB时自动清理

### 手动清理
如果需要手动清理日志文件：
1. 删除 `logs/` 目录下的旧日志文件
2. 或者使用日志清理工具

## 开发指南

### 在代码中添加日志

```python
from utils.logger import get_logger

# 获取日志实例
logger = get_logger("模块名")

# 记录不同级别的日志
logger.debug("调试信息")
logger.info("一般信息")
logger.warning("警告信息")
logger.error("错误信息")
logger.critical("严重错误")

# 记录异常
try:
    # 某些操作
    pass
except Exception as e:
    logger.error(f"操作失败: {e}")
    logger.exception("详细异常信息")
```

### 使用装饰器自动记录

```python
from utils.logger import log_function, log_exception

@log_function
def my_function(param1, param2):

    return result

    # 函数异常会自动记录
```

## 注意事项

1. **日志文件大小**: 长时间运行可能产生大量日志文件，注意磁盘空间
2. **敏感信息**: 避免在日志中记录密码、密钥等敏感信息
3. **性能影响**: 大量DEBUG日志可能影响程序性能，生产环境建议关闭
4. **文件权限**: 确保程序有权限创建和写入日志文件

## 故障排除

### 日志文件无法创建
- 检查 `logs/` 目录是否存在
- 检查程序是否有写入权限
- 检查磁盘空间是否充足

### 日志查看器无法打开
- 检查 `logs/` 目录是否存在
- 检查是否有可读的日志文件
- 查看程序错误信息

### 自动清理不工作
- 检查程序是否正常启动
- 查看日志中是否有清理相关的错误信息
- 手动检查日志文件是否被清理
